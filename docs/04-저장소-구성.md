## 1. egovframe-template-simple-backend

### 개요
- REST API 백엔드 템플릿
- JSP 없이 API만 제공

### 기술 스택
| 항목 | 버전 |
|------|------|
| Java | 1.8+ |
| Spring Boot | 2.x |
| Maven | 3.8.4 |
| MyBatis | - |
| Database | H2 (내장) → MySQL로 변경 |

### 주요 특징
- JavaConfig 기반 설정 (XML 없음)
- JWT 토큰 인증
- Swagger API 문서 자동 생성
- AOP 기반 트랜잭션
- REST API Controller 패턴

### 포함 기능
```
- 로그인 / 로그아웃 (JWT)
- 게시판 CRUD
- 회원 관리
- Swagger UI (/swagger-ui/index.html)
```

### 실행 방법
```bash
# CLI
mvn spring-boot:run

# JAR
java -jar <파일명>.jar --spring.profiles.active=<프로필>
```

## 2. egovframe-template-simple-react

### 개요
- React 기반 프론트엔드 템플릿
- 백엔드 API와 연동

### 기술 스택
| 항목 | 버전 |
|------|------|
| React | 18.x |
| Node.js | 18.12.0 |
| NPM | 8.19.2 |
| 빌드 도구 | Vite |
| Lint | ESLint |

### 주요 특징
- Vite 기반 빠른 빌드
- 컴포넌트 기반 구조
- ESLint 코드 품질 관리

### 포함 기능
```
- 메인 페이지
- 회원가입 / 로그인
- 소셜 로그인 (네이버, 카카오)
- 공지사항 / 게시판
- 갤러리
- 일정 관리
- 관리자 페이지
```

### 실행 방법
```bash
# 설치
npm install

# 개발 서버
npm run dev

# 빌드
npm run build
```

## 3. egovframe-common-components

### 개요
- 254개 공통 기능 컴포넌트
- 필요한 기능만 선택하여 적용

### 활용 방법
```
1. 필요한 컴포넌트 코드 분석
2. Service / DAO / Mapper 참고
3. 백엔드에 맞게 수정하여 적용
4. REST API로 변환
```

### CMS에 적용할 컴포넌트
| 패키지 | 컴포넌트 | 용도 |
|--------|---------|------|
| uat | 일반로그인 | JWT로 변환 적용 |
| sec | 권한관리, 롤관리 | 권한 체계 |
| uss/umt | 회원관리 | 회원 CRUD |
| cop/bbs | 게시판관리, 통합게시판 | 게시판 |
| cop/bbs | 댓글관리 | 댓글 |
| uss/olh | FAQ관리, Q&A관리 | 고객지원 |
| uss/ion | 팝업창관리, 배너관리 | 콘텐츠 |
| sts | 접속통계, 게시물통계 | 통계 |
| sym/mnu | 메뉴관리 | 메뉴 |
| sym/ccm | 공통코드 | 코드 |

## 개발 환경 셋팅 순서

### 1단계: 저장소 Clone
```bash
# 백엔드
git clone https://github.com/eGovFramework/egovframe-template-simple-backend.git backend

# 프론트엔드
git clone https://github.com/eGovFramework/egovframe-template-simple-react.git frontend

# 참고용 (선택)
git clone https://github.com/eGovFramework/egovframe-common-components.git reference/common-components
```

### 2단계: 백엔드 실행
```bash
cd backend
mvn spring-boot:run
# http://localhost:8080
# Swagger: http://localhost:8080/swagger-ui/index.html
```

### 3단계: 프론트엔드 실행
```bash
cd frontend
npm install
npm run dev
# http://localhost:3000
```

### 4단계: DB 변경 (H2 → MySQL)
```
1. pom.xml에 MySQL 드라이버 추가
2. application.properties 수정
3. 스키마/데이터 마이그레이션
```
