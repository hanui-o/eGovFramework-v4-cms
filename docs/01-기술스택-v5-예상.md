# 전자정부프레임워크 v5 기술스택 (예상)

> 출시 예정: 2025.12.05
> 본 문서는 v5 공식 발표 전 예상 내용입니다.

---

## 버전 비교

| 구분 | v4.3 (현재) | v5.0 (예상) |
|------|------------|------------|
| **Spring Framework** | 5.3.x | 6.x |
| **Spring Boot** | 2.x | 3.x |
| **Java** | 11+ | 17+ (필수) |
| **Jakarta EE** | javax.* | jakarta.* |

---

## Backend 예상

### 확정적 변화 (Spring 6 / Boot 3 요구사항)

| 항목 | v4.3 | v5 예상 |
|------|------|---------|
| **Java 최소 버전** | 11 | **17** (필수) |
| **패키지 네임스페이스** | javax.* | **jakarta.*** |
| **Spring Security** | 5.x | 6.x |
| **Hibernate** | 5.x | 6.x |

### 예상 변화

| 항목 | v4.3 | v5 예상 | 가능성 |
|------|------|---------|:------:|
| **빌드 도구** | Maven | Maven (기본), Gradle 지원 강화 | 높음 |
| **ORM** | MyBatis | MyBatis 유지 | 확실 |
| **DB 지원** | 8종 | 동일 유지 | 확실 |
| **Native Image** | 미지원 | GraalVM Native 지원? | 낮음 |

### 마이그레이션 주의점

```
1. javax → jakarta 패키지 변경 (전체 import 수정)
2. Spring Security 설정 방식 변경
3. 일부 deprecated API 제거
4. properties/yaml 설정 키 변경 가능
```

## v5 대비 전략

### 지금 할 수 있는 것

```
1. Java 17로 개발 환경 구성
2. Spring Boot 3 마이그레이션 가이드 숙지
3. jakarta.* 패키지 변경 이해
4. Vite + React 조합 연습
```

### v5 출시 후

```
1. 새 폴더에서 v5 템플릿으로 시작
2. 공통컴포넌트 마이그레이션 버전 확인
3. 기존 v4 코드 참고하여 빠르게 적용
```
